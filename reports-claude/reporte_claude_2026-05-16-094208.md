# Reporte Ecosistema Claude - 2026-05-16-094208

## Highlights del Día

1. **Claude Code v2.1.143** (15 mayo) agrega enforcement de dependencias entre plugins: `claude plugin disable` ahora rechaza deshabilitar si otro plugin activo depende del objetivo, y `claude plugin enable` habilita dependencias transitivas automáticamente. También incluye estimados de costo de contexto proyectados por turno en la vista de marketplace de plugins.
2. **Claude Agent SDK Python v0.2.82 y TypeScript v0.3.143** (15 mayo) introducen **cambio breaking**: los servidores MCP ahora se conectan en background por defecto. Las sesiones inician de inmediato y los servidores lentos reportan `status: "pending"` hasta estar listos. Se elimina la API v2 de sesiones en TypeScript (`unstable_v2_createSession`, etc.).
3. **Claude Platform on AWS** lanzado el 11 de mayo: acceso completo a la Messages API, Files API, Message Batches API, Managed Agents, Agent Skills, ejecución de código y tool use a través de endpoints nativos de AWS con billing e IAM de AWS.
4. **Fast mode para Claude Opus 4.7** disponible desde el 12 de mayo en research preview. Usa `speed: "fast"` con `model: "claude-opus-4-7"` y el header beta `fast-mode-2026-02-01` para generación de tokens de salida significativamente más rápida.
5. **SDK Python v0.102.0 y SDK TypeScript v0.96.0** (13 mayo) agregan soporte para `BetaManagedAgentsSearchResultBlock` y diagnósticos de caché beta, manteniéndose en sincronía con las nuevas capacidades de la API.

## Tendencias y Observaciones

1. **Aceleración en el ecosistema agéntico**: La semana del 11-15 de mayo muestra un ritmo intenso de releases centrados en agentes. Claude Code v2.1.143, Agent SDK Python v0.2.82 y Agent SDK TypeScript v0.3.143 se liberaron el mismo día (15 mayo), todos con cambios orientados a mejorar la gestión de plugins, dependencias y sesiones background. Esto sugiere una maduración coordinada del stack de agentes.

2. **Cambios breaking en MCP y gestión de sesiones**: La transición de MCP servers a modo de conexión en background por defecto (Agent SDK Python v0.2.82 y TypeScript v0.3.142) es un cambio significativo que afecta a todos los desarrolladores que consumen MCP servers síncronamente. La eliminación de la API v2 de sesiones en TypeScript (deprecated desde 0.2.133) consolida `query()` como interfaz única. Los equipos deben revisar sus integraciones.

3. **AWS como vector estratégico**: Claude Platform on AWS (11 mayo) y la apertura de Amazon Bedrock a todos los clientes (16 abril) reflejan una expansión agresiva hacia el ecosistema cloud empresarial. El SDK Python v0.101.0 y TypeScript aws-sdk-v0.3.0 agregan clientes AWS dedicados simultáneamente, lo que indica una estrategia de go-to-market coordinada.

4. **Ciclo de deprecaciones acelerado**: En 30 días se han retirado o deprecado Claude Haiku 3 (20 abril), la ventana de contexto beta 1M para Sonnet 4/4.5 (30 abril), y se han anunciado deprecaciones de Sonnet 4 y Opus 4 con retiro en junio 2026. Anthropic está consolidando activamente su lineup de modelos hacia las versiones 4.6/4.7.

5. **MCP ecosystem más estático**: Mientras los SDKs de Anthropic muestran releases semanales, los repos de modelcontextprotocol muestran baja actividad en el último mes (solo python-sdk tiene un release reciente). El specification y servers no tienen releases en meses, lo que puede indicar que el estándar MCP está en fase de estabilización o que el desarrollo activo se ha movido a implementaciones específicas.
